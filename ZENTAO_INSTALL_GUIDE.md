# 禅道集成配置指南

## 📋 概述

工作助手支持与禅道集成，实现任务和 Bug 的双向同步。您可以选择在安装时配置，也可以稍后在扩展设置中配置。

## 🚀 安装时配置（推荐）

### 一键安装脚本

运行安装脚本时，会自动引导您配置禅道：

```bash
./install.sh
```

### 配置步骤

#### 1. 选择是否配置禅道
```
🐉 禅道集成配置
禅道集成可以实现任务和 Bug 的双向同步

是否配置禅道集成？[y/N]:
```

- 输入 `y` 开始配置
- 输入 `N` 跳过（稍后可在扩展设置中配置）

#### 2. 输入禅道地址
```
禅道地址: http://10.128.1.8:8088
```

**格式要求：**
- 必须是完整的 URL，包含协议（http 或 https）
- 示例：`http://10.128.1.8:8088` 或 `https://zentao.example.com`

**如何获取禅道地址：**
- 联系您的禅道管理员
- 或查看浏览器中禅道的访问地址

#### 3. 输入用户名和密码
```
禅道用户名: your-account
禅道密码: ******
```

**注意：**
- 密码会明文保存在 `.env` 文件中
- 请确保 `.env` 文件的权限设置正确
- 建议使用专用的 API 账号，而不是个人账号

#### 4. 输入执行 ID
```
执行 ID (execution ID): 167
```

**如何获取执行 ID：**

1. 登录禅道
2. 进入要创建任务的执行
3. 点击"创建任务"
4. 查看 URL 中的数字

**示例：**
- URL: `http://zentao.example.com/zentao/task-create-167-0-0.html`
- 执行 ID: `167`

#### 5. 测试连接（可选）
```
是否测试禅道连接？[Y/n]: y
```

测试内容包括：
- ✓ 禅道服务器是否可访问
- ✓ 用户名和密码是否正确
- ✓ 是否能成功登录

**测试结果：**
- ✅ 成功：配置正确，可以使用
- ⚠️ 失败：配置已保存，请稍后手动测试或在扩展设置中重新配置

## 🔧 扩展设置中配置

如果在安装时跳过了禅道配置，可以在扩展设置中配置：

### 配置步骤

1. 打开工作助手扩展
2. 点击"设置"按钮
3. 找到"禅道集成"部分
4. 填写以下信息：
   - 禅道地址
   - 禅道账号
   - 禅道密码
   - 执行 ID
5. 点击"测试连接"验证配置
6. 点击"保存配置"

### 测试连接

点击"🧪 测试连接"按钮，验证配置是否正确：
- ✓ 连接成功：配置正确
- ❌ 连接失败：请检查配置信息

## 🎯 使用禅道集成

### 功能说明

配置完成后，可以实现以下功能：

#### 1. 任务同步
- 在工作助手中创建的任务自动同步到禅道
- 支持拖拽调整状态
- 状态变更自动同步到禅道

#### 2. Bug 管理
- 创建 Bug 并同步到禅道
- Bug 状态管理（未确认 → 已激活 → 已解决 → 已关闭）
- 自动同步到禅道

#### 3. 用户管理
- 自动获取禅道用户列表
- 方便指派任务和 Bug

#### 4. 工时记录
- 自动计算工作时长
- 自动记录到禅道

## ❓ 常见问题

### 1. 连接测试失败

**可能原因：**
- 禅道地址不正确
- 网络连接问题
- 用户名或密码错误
- 禅道服务未运行

**解决方法：**
1. 检查禅道地址是否正确
2. 确认网络连接正常
3. 验证用户名和密码
4. 联系禅道管理员

### 2. 执行 ID 获取不到

**解决方法：**
1. 登录禅道
2. 进入"执行"或"项目"视图
3. 选择对应的执行
4. 点击"创建任务"
5. 从 URL 中复制数字

### 3. 密码安全

**建议：**
- 使用专用 API 账号
- 定期更换密码
- 设置 `.env` 文件权限为 `600`
  ```bash
  chmod 600 .env
  ```

### 4. 多个执行 ID

**解决方案：**
- 扩展支持配置"常用执行"
- 可以在扩展设置中添加多个执行 ID
- AI 会优先匹配常用执行

## 🔒 安全建议

1. **使用专用账号**
   - 为工作助手创建专用的禅道账号
   - 限制该账号的权限

2. **文件权限**
   ```bash
   chmod 600 .env  # 只有所有者可读写
   ```

3. **定期更换密码**
   - 定期更换禅道密码
   - 及时更新 `.env` 文件

4. **不要提交到版本控制**
   - 确保 `.env` 文件不被 Git 追踪
   - 已在 `.gitignore` 中排除
