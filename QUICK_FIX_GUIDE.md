# 🔧 Bug 创建失败 - 快速修复指南

## ❌ 你遇到的问题

```
[BugManager] ✗ 获取 productID 失败: Frame with ID 0 is showing error page
```

这个错误表示禅道标签页显示的是错误页面，而不是正常的执行页面。

---

## ✅ 快速解决方案

### 方案 1: 手动验证禅道页面（推荐先做）

#### 步骤 1: 手动访问执行页面
在浏览器中打开：
```
http://10.128.1.8:8088/zentao/execution-bug-167.html
```

#### 步骤 2: 检查页面状态

**如果页面正常显示** → 继续方案2

**如果页面显示错误**：
- **404错误**: 执行ID 167 不存在，需要使用正确的执行ID
- **403错误**: 没有权限，联系管理员
- **登录页面**: 需要重新登录禅道
- **其他错误**: 检查网络连接

### 方案 2: 检查执行配置

#### 1. 确认正确的执行ID

1. 登录禅道
2. 进入 **执行** 或 **项目** 页面
3. 查找你想使用的执行
4. 记录正确的执行ID

#### 2. 更新执行配置

1. 打开工作助手扩展
2. 点击 **设置** → **执行管理**
3. 选择正确的执行
4. 点击 **设为默认**

### 方案 3: 重新登录禅道

1. **在禅道标签页重新登录**
2. **刷新扩展**:
   - 打开 `chrome://extensions/`
   - 找到 **工作助手**
   - 点击 **🔄 重新加载** 按钮

### 方案 4: 应用代码修复（如果上述方案无效）

我已经为你准备了详细的修复文档和代码补丁：

#### 📄 文档位置
- **详细分析**: [docs/BUG_CREATE_ERROR_TROUBLESHOOTING.md](docs/BUG_CREATE_ERROR_TROUBLESHOOTING.md)
- **代码补丁**: [bug-create-error-fix.patch](bug-create-error-fix.patch)

#### 🔧 应用修复
1. 备份原始文件
2. 查看 `bug-create-error-fix.patch` 中的代码
3. 按照说明应用修复
4. 重新加载扩展

## 🎯 预防措施

### 1. 定期检查登录状态

禅道 Session 每隔几小时会过期，建议：
- 每天第一次使用前重新登录禅道
- 如果长时间未使用，先登录再创建Bug

### 2. 验证执行配置

创建Bug前，确认：
- ✅ 执行ID正确
- ✅ 有该执行的访问权限
- ✅ 执行状态为"进行中"

### 3. 保持扩展更新

定期检查扩展更新：
- 打开 `chrome://extensions/`
- 启用"开发者模式"
- 点击"更新扩展"

## 📋 常见问题 FAQ

### Q1: 为什么会出现这个错误？

**A**: 主要原因有：
1. 禅道 Session 过期，需要重新登录
2. 执行ID错误或不存在
3. 没有该执行的访问权限
4. 网络连接问题

### Q2: 如何找到正确的执行ID？

**A**:
1. 登录禅道
2. 进入 **执行** 或 **项目** 页面
3. 点击你想使用的执行
4. 查看浏览器地址栏，格式如：
   ```
   http://10.128.1.8:8088/zentao/execution-view-167.html
   ```
   其中 `167` 就是执行ID

### Q3: 应用代码补丁会丢失数据吗？

**A**: 不会。代码补丁只是修改错误处理逻辑，不会影响你的任务和Bug数据。

### Q4: 修复后还需要做什么？

**A**:
1. 重新加载扩展
2. 重新登录禅道
3. 创建测试Bug验证修复
